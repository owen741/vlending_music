## 🔄 업데이트 절차

### 단계 1: 이미지 파일 추가/교체

#### 새 이미지 추가 시:
1. 피그마에서 이미지 내보내기 → 파일명 확인
2. `src/assets/` 폴더에 파일 추가
3. `vite.config.ts`에 alias 추가
4. `src/vite-env.d.ts`에 타입 선언 추가

#### 기존 이미지 교체 시:
1. 피그마에서 같은 파일명으로 내보내기
2. `src/assets/` 폴더의 기존 파일 교체
3. 별도 설정 불필요 (alias는 파일명 기반)

### 단계 2: 코드 업데이트
- [ ] 새 이미지 경로를 코드에 반영
- [ ] 스타일 변경사항 반영
- [ ] 레이아웃 변경사항 반영

### 단계 3: 테스트
- [ ] `npm run dev` 실행하여 확인
- [ ] `npm run build` 실행하여 빌드 확인
- [ ] 모든 페이지에서 이미지 정상 표시 확인

## ⚠️ 주의사항

1. **파일명은 절대 수동으로 변경하지 말 것**
   - Figma에서 내보낸 파일명을 그대로 사용

2. **기존 alias는 보존**
   - 새 이미지 추가 시 기존 alias는 그대로 유지

3. **타입 선언과 alias는 항상 쌍으로 관리**
   - `vite.config.ts`에 alias 추가 시 → `vite-env.d.ts`에도 타입 선언 추가

4. **GitHub Pages 배포 설정은 건드리지 말 것**
   - `base: '/vlending_music/'` 경로 유지
